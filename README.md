# UPI Transaction Analytics
### End-to-End Data Analytics Pipeline

---

## Overview
End-to-end data analytics project analyzing 100,000 UPI 
transactions to detect fraud patterns, identify transaction 
failure root causes, and deliver strategic business 
recommendations using Excel, MySQL, Python and Power BI.

---

## Tech Stack
MySQL | Python | pandas | scipy | scikit-learn | 
matplotlib | seaborn | Power BI | SQLAlchemy | Excel

---

## Dataset
7 relational tables — 100,000 transactions, 10,000 customers,
500 merchants, 621 fraud alerts | Time Period: 2021–2025

> Note: Raw data not included due to size constraints.

---

## Project Workflow
Business Problem → Excel Exploration → MySQL Schema Design 
→ Data Ingestion → Python Cleaning → Statistical Analysis 
→ Power BI Dashboards → Business Recommendations

---

## Key Insights
1. Feature phones show highest fraud rate (~2% above average)
2. Incorrect PIN drives 5.87% overall transaction failure rate
3. August shows critical fraud spike of 400+ alerts
4. Top 3 merchants flagged as high risk — immediate audit needed
5. P2P transactions lack merchant-level fraud controls

---

## Strategic Recommendations
- R1 — 2FA for feature phones → reduce fraud 2% to below 1% | 6 months
- R2 — Audit top 10 merchants → fraud ratio below 2% | 3 months
- R3 — Customer PIN education → failure rate below 3% | 3 months
- R4 — Real-time fraud monitoring → alerts down 40% | 6 months
